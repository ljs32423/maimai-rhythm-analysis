# maimai-rhythm-analysis 项目记忆

## 项目概述
这是一个 maimai 谱面节奏解析可视化项目。当前产物是：
- 节奏解析图片：`{难度}_rhythm.svg`、`{难度}_rhythm.png`
- 网页滚动条 SVG：`{难度}_strip.svg`
- 谱面预览网页：`{难度}_analysis.html`
- 谱面预览视频：`{难度}_preview.mp4`
- 音频对齐偏移：`{难度}_offset.txt`

当前项目位置：
`E:\Code\maimai-rhythm-analysis`

## songs 目录规则
- 所有歌曲统一放在 `songs\` 下。
- 首次扫描旧目录时，会自动把旧歌曲目录迁移到 `songs\`。
- 歌曲目录名会按 `maidata.txt` 里的 `&title` 自动重命名。
- Windows 非法文件名字符会替换为全角字符。
- 选择歌曲时兼容旧目录名、当前曲名、绝对路径。

## 难度与文件名规则
- 界面显示名：
  - `5 -> MASTER`
  - `6 -> Re:MASTER`
- 磁盘文件名前缀：
  - `5 -> MASTER`
  - `6 -> ReMASTER`
- `Re:MASTER` 预览视频查找兼容这些别名：
  - `ReMASTER`
  - `REMASTER`
  - `Re-MASTER`
  - `Re_MASTER`
  - `Re MASTER`
  - `Re：MASTER`

## 默认处理难度
- 不指定 `-diff` 时，默认只处理：
  - `MASTER`
  - `Re:MASTER`（如果存在）
- 如果一首歌没有这两个难度，才回退到该曲实际已有难度。
- 这条规则已统一应用到：
  - `visualize.py`
  - `render_preview.py`
  - `align_audio.py`
  - `make_html.py`
  - `run_all.py`

## simai_parser.py
- 已修正时值换算问题：
  - comma：`240 / (BPM * division)`
  - `[a:b]`：`240 * b / (a * BPM)`
- 支持：
  - TAP / HOLD / BREAK / EX / SLIDE / TOUCH / TOUCH_HOLD / FIREWORK
  - BPM 时间线 `chart.bpm_timeline`
  - `{#sec}` 自由时长
  - `E` 触摸传感器
  - 连接滑条、同始点滑条、大 V 等

## visualize.py
- 节奏点配色和形状由“最近的前/后节奏点”决定，不再固定看后一个点。
- 时值文字仍按“到下一个节奏点”的时值显示。
- 不显示拍号，不显示小节数。
- 不显示旋转六边形，不显示右侧装饰五边形。
- 折叠 PNG 不额外补尾部小节。
- 长条 SVG / 网页 strip SVG 会额外补尾部小节，避免播放到尾部时底部全黑。
- 当前主要几何参数：
  - `PX_PER_BEAT = 100`
  - `NOTE_R = 12.0`
  - `NOTE_AREA_H = 44`
  - `LABEL_AREA_H = 14`
  - `LABEL_GAP = 8`
- 时值字号已调大，并加入相邻标签压缩，避免重叠。

## 变 BPM
- `simai_parser.py` 会把每个 `(BPM)` 记录进 `chart.bpm_timeline`
- `time_to_beat()` 按分段积分做秒到拍的换算
- SVG / PNG 会绘制黄色 BPM 变化标记
- HTML 滚动速度按当前时间所属 BPM 段实时计算
- 左侧 BPM 数字会随播放实时更新

## render_preview.py
- 当前正式使用 MajdataView 自动录制谱面预览视频。
- 默认录制参数：
  - Tap 流速：`7.5`
  - Touch 流速：`7.5`
  - PV 遮罩：`0.5`（50% 亮度）
  - 分辨率：`3840x2160`
  - 帧率：`60 FPS`
- `render_preview.py` 顶部注释提供 Full HD、2K、4K 三档分辨率，修改常量即可切换。
- 为避免 PV 卡顿，录制前会在临时目录将 PV 转成恒定 60 FPS、零起始时间戳的 H.264 Baseline 视频；原始 PV 不会被覆盖。
- MajdataView 实时编码使用 `ultrafast` / `zerolatency`，降低 4K 录制时的帧管道阻塞。
- 键音默认开启：`ENABLE_KEY_SOUNDS = True`，默认音量 `KEY_SOUND_VOLUME = 0.7`。
- MajdataView HTTP 录制接口本身没有键音字段；项目按 MajdataEdit 官方事件规则，将 `SFX\` 中的 Answer、Judge、Break、EX、Slide、Touch、Touch Hold、烟花等音效混入临时 `out.wav`。
- 内部流程：
  - 安装 MajdataView
  - 通过 `tools\src\majdata_bridge` 的源码构建 Bridge，并用运行产物把 `maidata.txt` 转成 `majdata.json`
  - 通过 MajdataView HTTP API 触发录制
  - 校验输出视频时长和画面是否非纯黑

## align_audio.py
- 对齐锚点：
  - 谱面预览视频中第一个键到达判定线的时间
  - 必须与滚动条中第一个音符到达判定圆的时间一致
- 不是手工逐曲调偏移，而是每次自动计算。
- 策略：
  - 优先用 `track.mp3` 和预览视频整曲音轨做能量包络相关匹配
  - 再用波形局部相关细化
  - 置信度不足时回退到首个持续高能量 onset 检测
- 输出到 `{难度}_offset.txt`

## make_html.py
- 当前页面结构：
  - 上方：谱面预览视频
  - 下方：Arcaea 风格节奏滚动条
- 现状：
  - 左侧保留三层五边形 BPM 承载区
  - 右侧不做装饰边框
  - 不显示拍号
  - 不显示小节数
  - 不显示旋转六边形
- 控制条位置已经调整：
  - 从视频底部移到视频右上角
  - 避免挡住谱面预览下沿
- 同步公式核心：
  - `chartT = videoT - VIDEO_OFFSET + userOffset`
  - 变 BPM 下按分段时间线算当前拍位

